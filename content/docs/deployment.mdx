# Deployment

OSC runs on **Kubernetes** across three environments. All manifests live in the `censof-k8s-config` repository.

```
censof-k8s-config/
├── docker-desktop/          ← Local single-cluster (your machine)
├── development/             ← 3 remote clusters (dev servers)
│   ├── cluster1-backend/
│   ├── cluster2-fe-admin/
│   └── cluster3-fe-public/
└── production/              ← 3 remote clusters (HA + autoscaling)
    ├── cluster1-backend/
    ├── cluster2-fe-admin/
    └── cluster3-fe-public/
```

---

## Environment Comparison

| | Docker Desktop | Development | Production |
|---|---|---|---|
| **Clusters** | 1 (local) | 3 (remote servers) | 3 (remote servers) |
| **Images** | Local (`imagePullPolicy: Never`) | GitLab registry (`:dev` tags) | GitLab registry (`:latest` tags) |
| **Replicas** | 1 | 1 | 2 + HPA (2-5 pods, 70% CPU) |
| **DB / Redis** | Bundled in K8s (`database` namespace) | External IPs (`10.29.46.108` / `10.29.46.109`) | External IPs |
| **Ingress** | Traefik (`*.test` domains) | NGINX (`dev-kpkt-api.test`) | NGINX + TLS (cert-manager) |
| **TLS** | No | No | Yes |
| **Health Checks** | No | No | Yes |
| **Resource Limits** | 100m CPU / 128Mi | 100m CPU / 128Mi | 500m CPU / 512Mi |
| **Queue Workers** | Yes (08-workers.yaml) | Yes (workers.yaml) | Not in manifests |
| **Email** | SMTP4Dev (local) | SMTP4Dev (remote) | External SMTP |

## Local Deployment (Docker Desktop)

This is the recommended way to run the full OSC stack on your machine.

### Prerequisites

1. **Docker Desktop** installed with **Kubernetes enabled** (Settings → Kubernetes → Enable)
2. **kubectl** available in your terminal (`docker desktop` bundles it)
3. **Traefik** ingress controller (or use port-forwarding instead)
4. All **source code repositories** cloned locally

### Step 1: Build All Docker Images

Each service has a `Dockerfile`. Build them with the exact image names the manifests expect:

```powershell
# Backend services (from each repo root)
docker build -t be0:latest ./Backend/osc-be0-utiliti-main
docker build -t be1:latest ./Backend/osc-be1-admin-profile-main
docker build -t be2:latest ./Backend/osc-be2-pelesenan-main
docker build -t be3:latest ./Backend/osc-be3-aduan-dan-notifikasi-main
docker build -t be4:latest ./Backend/osc-be4-reporting-main
docker build -t be5:latest ./Backend/osc-be5-enforcement-main
docker build -t be6:latest ./Backend/osc-be6-chatbot-main

# Frontend services
docker build -t fe-admin:latest  ./Frontend/osc-fe-admin-main
docker build -t fe-public:latest ./Frontend/osc-fe-customer-main
```

:::warning
Image names must **exactly match** what the manifests use (`be0:latest`, `be1:latest`, etc.). The docker-desktop manifests set `imagePullPolicy: Never` so Kubernetes will only use local images.
:::

### Step 2: Apply K8s Manifests (In Order)

```powershell
cd Infra/censof-k8s-config-clone/docker-desktop

# 1. Create namespaces (backend, frontend, database)
kubectl apply -f 00-namespaces.yaml

# 2. ConfigMaps and Secrets (DB credentials, API URLs, Redis config)
kubectl apply -f 01-configmaps.yaml

# 3. MySQL 8.0 (creates osc_pelesenan DB + osc_user)
kubectl apply -f 02-mysql.yaml

# 4. Redis 7
kubectl apply -f 03-redis.yaml

# 5. All 7 backend services (be0-be6)
kubectl apply -f 04-backends.yaml

# 6. Both frontends (fe-admin, fe-public)
kubectl apply -f 05-frontends.yaml

# 7. Ingress routes (optional — can use port-forward instead)
kubectl apply -f 06-ingress.yaml

# 8. SMTP4Dev for email testing
kubectl apply -f 07-smtp4dev.yaml

# 9. Queue workers + scheduler
kubectl apply -f 08-workers.yaml
```

Or apply everything at once:

```powershell
kubectl apply -f .
```

### Step 3: Wait for Pods

```powershell
# Watch all pods come up
kubectl get pods -A -w

# Expected: ~15-20 pods across 3 namespaces
kubectl get pods -n database     # mysql, redis, smtp4dev
kubectl get pods -n backend      # be0-be6 + queue workers + scheduler
kubectl get pods -n frontend     # fe-admin, fe-public
```

### Step 4: Run Database Migrations

Once MySQL is running, exec into a backend pod to run migrations:

```powershell
# Find a running backend pod
kubectl exec -it -n backend deployment/be0 -- php artisan migrate --force

# Seed initial data (if applicable)
kubectl exec -it -n backend deployment/be0 -- php artisan db:seed --force
```

### Step 5: Access the Application

<Tabs items={['Port Forward (Simplest)', 'Ingress with Traefik', 'NodePort']}>
  <Tab value="Port Forward (Simplest)">

No hosts file changes needed:

```powershell
# Admin portal → http://localhost:8080
kubectl port-forward -n frontend svc/fe-admin 8080:80

# Customer portal → http://localhost:8081
kubectl port-forward -n frontend svc/fe-public 8081:80

# MySQL direct access → localhost:3306
kubectl port-forward -n database svc/mysql 3306:3306

# SMTP4Dev web UI → http://localhost:8025
kubectl port-forward -n database svc/smtp4dev 8025:80
```

  </Tab>
  <Tab value="Ingress with Traefik">

The manifests use `ingressClassName: traefik`. Install Traefik first:

```powershell
# Install Traefik via Helm
helm repo add traefik https://traefik.github.io/charts
helm install traefik traefik/traefik
```

Then add to `C:\Windows\System32\drivers\etc\hosts`:

```

127.0.0.1  fe-public.test
127.0.0.1  api.utilities.test
127.0.0.1  api.profile.test
127.0.0.1  api.license.test
127.0.0.1  api.complaints.test
127.0.0.1  api.reporting.test
127.0.0.1  api.enforcement.test
127.0.0.1  api.chatbot.test
```

Access:
- Admin: `http://fe-admin.test`
- Customer: `http://fe-public.test`
- BE0 API: `http://api.utilities.test`

### What Gets Deployed

| Namespace | Pods | Purpose |
|-----------|------|---------|
| `database` | mysql, redis, smtp4dev | MySQL 8.0 (`osc_pelesenan`), Redis 7 (cache/session/queue), SMTP4Dev (email) |

| MySQL (root) | `root` | `local_dev_password` |
| MySQL (app) | `osc_user` | `osc_password` |
| Redis | — | No password |
| Database name | — | `osc_pelesenan` |

## Alternative: Docker Compose (Per-Service)

Each service also has its own `docker-compose.yml` for running individually. This requires a shared Docker network:

```powershell
# Create the shared network first
docker network create osc-network

# Then start individual services
cd Backend/osc-be0-utiliti-main
docker-compose up -d    # Starts on port 9000

cd ../../Backend/osc-be1-admin-profile-main
docker-compose up -d    # Starts on port 9000

cd ../../Frontend/osc-fe-admin-main
docker-compose up -d    # Starts on port 8080

cd ../../Frontend/osc-fe-customer-main
docker-compose up -d    # Starts on port 8081
```

:::info
The docker-compose approach runs services outside Kubernetes. Each service connects to `osc-mysql` and `osc-redis` on the `osc-network` bridge network. You'll need MySQL and Redis containers running on that network separately.
:::

## Production Environment (HA Clusters)

Same 3-cluster topology as development, but with:
- **2 base replicas** per service
- **HPA** autoscaling to 5 pods at 70% CPU
- **TLS** via cert-manager
- **Individual YAML files** per backend (`be0.yaml`...`be6.yaml`)
- **500m CPU / 512Mi** resource limits

### Deploy to Production

```bash
# Cluster 1 — Backend
kubectl apply -f production/cluster1-backend/00-namespace.yaml
kubectl apply -f production/cluster1-backend/be0.yaml
kubectl apply -f production/cluster1-backend/be1.yaml
# ... through be6.yaml
kubectl apply -f production/cluster1-backend/ingress.yaml

# Cluster 2 — FE Admin
kubectl apply -f production/cluster2-fe-admin/

# Cluster 3 — FE Public
kubectl apply -f production/cluster3-fe-public/
```

### Verify Production

```bash
# Check HPA is active
kubectl get hpa -n backend
kubectl get hpa -n frontend

# Check pod distribution across nodes
kubectl get pods -o wide -n backend

# Check TLS certificates
kubectl get certificates -n frontend
```

## Troubleshooting

<Tabs items={['Pod Issues', 'Database', 'Networking', 'Images']}>
  <Tab value="Pod Issues">

```powershell
# Check pod status and events
kubectl describe pod -n backend -l app=be0

# View logs
kubectl logs -n backend deployment/be0

# View logs for crashed pod
kubectl logs -n backend deployment/be0 --previous

# Restart all pods in a namespace
kubectl rollout restart deployment -n backend
```

  </Tab>
  <Tab value="Database">

```powershell
# Check MySQL pod
kubectl get pods -n database -l app=mysql

# Connect to MySQL directly
kubectl exec -it -n database deployment/mysql -- mysql -uosc_user -posc_password osc_pelesenan

# Run migrations from any backend pod
kubectl exec -it -n backend deployment/be0 -- php artisan migrate --force

# Check Redis
kubectl exec -it -n database deployment/redis -- redis-cli ping
```

  </Tab>
  <Tab value="Networking">

```powershell
# Test internal DNS resolution from a pod
kubectl exec -it -n backend deployment/be0 -- nslookup mysql.database.svc.cluster.local

# Test connectivity to another service
kubectl exec -it -n frontend deployment/fe-admin -- curl -s http://be1.backend.svc.cluster.local/health

# Check ingress config
kubectl get ingress -A
kubectl describe ingress -n frontend
```

  </Tab>
  <Tab value="Images">

```powershell
# Check if image exists locally (Docker Desktop)
docker images | Select-String "be0|be1|fe-admin"

# Rebuild and restart
docker build -t be0:latest ./Backend/osc-be0-utiliti-main
kubectl rollout restart -n backend deployment/be0

# Force pull new image (dev/prod only)
kubectl set image -n backend deployment/be0 be0=10.29.46.111:5050/development/backend-be0:dev
kubectl rollout restart -n backend deployment/be0
```

## Cleanup

```powershell
# Delete everything (Docker Desktop)
kubectl delete -f docker-desktop/

# Or delete by namespace
kubectl delete namespace backend frontend database

# Docker Compose cleanup
docker-compose down -v   # per-service
docker network rm osc-network
```
