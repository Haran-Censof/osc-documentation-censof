## What's Running

### Backend Services (7 microservices)

| Service | What it does |
|---------|--------------|
| **be0** | Utilities & shared functions |
| **be1** | User profiles & admin |
| **be2** | Licensing |
| **be3** | Complaints & notifications |
| **be4** | Reports |
| **be5** | Enforcement |
| **be6** | Chatbot |

All services:
- Listen on port 8080 (prod) or 80 (dev)
- Run as ClusterIP services (internal only, accessed via ingress)
- Have `/health` endpoint for readiness + liveness checks

### Frontend Services (2 apps)

- **fe-admin**: Admin dashboard
- **fe-public**: Customer portal

Both on port 80, NodePort in Docker Desktop, ClusterIP in prod/dev.

### Infrastructure (Docker Desktop only)

- **MySQL 8.0**: database `osc_pelesenan`, user `osc_user`
- **Redis 7**: cache & sessions & queues
- **SMTP4Dev**: local email testing


## Background Jobs (Development only)

Queue workers + scheduler run in dev. Production has none (⚠️ question mark).

- **Queue workers** (be0-queue...be6-queue): `php artisan queue:work --sleep=3 --tries=3 --max-time=3600`
- **Scheduler** (be0-scheduler): `php artisan schedule:work`

## 🚨 Important Things to Check

### 1. **Frontend images pointing to wrong place in prod**

### 3. **No Redis password**
`REDIS_PASSWORD=""` everywhere. Works if Redis is network-isolated, but risky.

### 4. **Database passwords in git**

### 5. **GitLab registry secret missing**

### 6. **No queue/scheduler in production**
Workers and scheduler only exist in dev. How do async jobs run in prod?

### 7. **Production DB config unknown**
No MySQL deployment in prod folder. Is it external? Where's the ConfigMap with prod DB_HOST?

### 8. **Different env variable names**
Dev uses `CACHE_STORE`, Docker uses `CACHE_DRIVER`. Suggests different app versions?

### 9. **No pod spreading or affinity rules**
If 2 replicas end up on same node, you lose HA. No rules prevent this.
